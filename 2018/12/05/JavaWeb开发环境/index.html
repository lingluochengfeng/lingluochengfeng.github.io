JavaWeb开发环境

Posted on 2018-12-05 | In JavaWeb

Java Web开发环境

  在开发Java Web应用程序之前，首先需要搭建Java Web的开发和运行环境。

  • JDK（Java Development kit）：Sun官方的Java开发和运行环境。
  • Eclipse、MyEclipse或者Idea：目前流行的Java Web集成开发环境（Integrated Develop Environment,简称IDE）
  • Tomcat：开源Web应用服务器，也是应用最广的Java服务器。

开发工具包JDK

  要运行Java程序，首先要安装Java运行环境，即JRE（Java Runtime Environment）。但作为开发人员，除了需要JRE以外，还需要安装Java的开发环境，即JDK。JDK=JRE+开发工具集（javac等），JRE=JVM（Java Virtual Machine，Java虚拟机）+JavaSE标准类库。

JDK

下载JDK

  到sun（oracle）官网下载最新的JDK，然后安装即可使用，安装完jdk之后会提醒你是否安装JRE，要选择“是”。

配置环境变量

  安装完JDK之后需要配置环境变量。
  在MAC下配置Java环境变量可以参考这篇文章:MAC安装JDK及环境变量配置
  在Windows下配置Java环境变量可以参考菜鸟教程的这篇文章:Windows 10 配置Java 环境变量

IDE

下载IDE

  选择一款自己喜欢的Java编译器，现在流行的有Eclipse、MyEclipse和Idea。Eclipse是很老且一直很流行的的一款编译器，占有大部分市场，我个人觉得Eclipse和MyEclipse差别不大，Eclipse是免费的，而MyEclipse则需要收费，还要自己去找破解方法。Idea是最近几年新流行起来的编译器，其代码提示、项目组织结构比Eclipse更加强大，对Git、Tomcat、Maven等提供了更好的支持，而且提供了对Eclipse快捷键的支持，可以方便程序员进行开发选择。

Tomcat

  Tomcat是一个免费，开源的Java Web服务器，属于轻量级应用服务器，在中小型系统和并发访问用户不是很多的场合下被普遍使用，是开发和调试JSP的首选。Java Web服务器在JavaEE规范中也被称为容器（Container），因为所有的Web应用程序都要部署到容器中运行。
   Apache基金会有两款常用软件：Apache web服务器和Tomcat web服务器

  • Apache web服务器专门处理HTML页面
  • Tomcat web服务器，不仅可以处理servlet和jsp，而且也能处理html页面，但不如Apache web服务器。

  • 在开发中，一般使用Tomcat处理servlet和jsp，使用Apache服务器处理html页面，两者经常被整合在一起使用。
    注意：

  • 下载安装Tomcat之前，需要安装JDK，并配置Java环境变量

  • 安装版（给客户使用）
  • 解压版：解压当前文件夹，目录不能有中文
  • 启动服务器：Tomcat路径/bin/startup.bat(启动服务器)
  • 访问：http://localhost:8080
  • 关闭服务器：点击关闭按钮，或者执行shutdown.bat

下载Tomcat

  前往Apache的官网:https://tomcat.apache.org/index.html下载不同操作系统所对应的Tomcat版本，Zip版本的Tomcat不需要安装，直接解压并配置环境变量即可使用。

解压配置Tomcat

  将下载到的Tomcat的zip文件解压到任意文件夹，解压完成后添加环境变量，添加环境变量可以参考这篇文章：Tomcat安装与环境变量的配置

常见启动问题

配置环境变量

  windows下启动一闪而过，环境变量没有配置好，要分别配置Java环境变量和Tomcat环境变量。

端口占用问题

  java.net.BindException:Address already in use:JVM_Bind
  端口被占用

解决方法：
  • 结束掉占用端口的应用程序（找到占用端口的应用程序，在控制台通过命令 netstat -ano,在任务管理器中结束程序）
  • 修改端口号（修改配置文件，Tomcat路径下的conf/server.xml配置文件）
  • 有应用程序占用80端口，使用上述方式解决
  • 有的WIN7系统自带World wide web publish IIS服务，默认占用80端口，如果有，去服务中，改为手动启动。

设置Tomcat端口号

  HTTP协议默认的端口号为80。如果服务器端口为80，则URL中的端口号可以省略，否则必须使用冒号加端口号指明端口。大部分的Web服务器都是80端口的。而Tomcat默认的端口号是8080，而非默认端口，因此在URL中必须指明端口（即：http://localhost:8080）
  Tomcat的端口号可以在Tomcat路径下的/conf/server.xml中配置。文件中的Connector标签中配置了端口号：

  在port中可以修改为80或其他端口号。建议修改文件前先备份该文件。如果指定的端口号与系统中其他程序冲突，则Tomcat会启动失败。开发模式下建议使用默认端口号8080

设置临时域名

  由于Tomcat服务器在本机上，因此我们可以通过localhost来访问。localhost表示本机。访问Tomcat服务器的方式有：

  • 使用域名localhost访问。仅限于本机，如：http://localhost:8080
  • 使用IP地址127.0.0.1访问。仅限于本机，例如：http://127.0.0.1:8080。本机IP为保留IP，代表本机。
  • 使用机器名称访问。仅限于本机或者局域网内。例如：http://机器名称:8080
  • 使用本机IP进行访问。例如：http://IP:8080
  • 如果有域名指向该服务器，可以使用域名访问。

  除了以上的方式以外，Windows还提供了一种机制，为机器设置临时域名。打开C:\WINDOWS\system32\drivers\etc文件夹，编辑下面的文件hosts。在文件的最后添加一个IP地址与对应的主机名称，如：127.0.0.1 www.google.com。表明这个域名指向了该服务器，启动Tomcat之后，可以通过www.google.com:8080访问Tomcat服务器。

不建议修改host文件。仅用于用某域名来测试本机上的Web程序。

Tomcat目录结构

  • bin(**) 启动项(startup.bat)，关闭项(shuodown.bat)
  • conf(**) 里面Tomcat配置文件(核心配置文件：server.xml，用户权限配置文件：tomcat-users.xml，所有web项目默认配置文件：web.xml)
  • lib(**) 服务器运行使用的jar包
  • logs(**) 日志文件，运行时产生的日志(localhost_access_log.\.txt   tomcat记录用户访问信息，星*表示时间)
  • temp(**) 运行时临时文件,文件夹内容可任意删除
  • webapps(**) web applications(web应用)
  • work(**) JSP翻译成Servlet程序

第一个Java Web程序

新建一个Web Project项目

  新建Web项目后，项目左侧的导航视图会显示项目结构。

  web.xml为编译器自动生成的程序描述文件(Application Descriptor)

新建Servlet

  在项目下的src文件夹下右击新建一个Servlet，新建一个Servlet之后，编译器会自动将Servlet相关的配置文件添加到web.xml文件中。

  这是web.xml自动添加的内容。servlet指明了Servlet类路径以及名称。servlet-mapping指明了访问指定名称的servlet的URI。此时，我们启动Tomcat之后，可以通过：http://localhost:8080/项目名/servlet/HelloServlet进行访问Tomcat。
  访问的内容就是Servlet中的内容，这里我们输入一句话，访问Tomcat的这个网址时，会显示这句话

  默认都是get方式进行访问，post访问方式以后会介绍。也可以不通过手动创建Servlet，可以添加一个继承自HTTPServlet的类，然后把Servlet配置到web.xml中。

部署到Tomcat

  部署(Deploy)是指将程序部署到Tomcat下，也就是Web程序的发布。部署时需要部署所有的class文件、web.xml、jar文件、JSP文件等。

手动部署

  在Tomcat目录的webapps下面新建文件夹，取名为firstWeb，然后找到项目工作目录，将该目录firstWeb\WebRoot下的所有内容复制到Tomcat目录下刚刚建好的webapps\firstWeb下，这样就完成了手工部署firstWeb的部署。然后启动Tomcat就可以访问:http://localhost:8080/firstWeb/servlet/HelloServlet访问上面新建的Servlet。

自动部署

  在Eclipse、MyEclipse和Idea中都可以对Tomcat进行相关配置，配置完成之后，直接启动Tomcat，然后输入网页URL即可直接访问。

导出为WAR包

  可以将程序导出为WAR文件。WAR文件就是普通的ZIP文件，只是里面包含一个Web应用程序。WAR的全称为Web Archive。

调试程序DEBUG模式

  调试（Debug）是程序开发中非常有用的一个工具，使用调试可以分步地执行程序，观察程序执行中的变量值以及执行流程。
  DuBug的功能：

  • 打断点，选择DeBug As
  • Step Over 跳过
  • Resume 执行下一个断点，如果没有断点，执行完成。
  • Step Into 跳入
  • Step Return 跳出
  • Drop To Frame 返回最开始的位置

  （1）调试前确定Tomcat的启动是不是Debug模式。在要调试的代码行左侧双击鼠标设置一个断点（Breakpoint），打好断点之后，使用Debug As运行项目。（再次双击代码行左侧取消断点）
  （2）在浏览器中输入要访问的网址，程序运行到断点处会暂停运行。
  （3）此时会转换到Debug视图，程序暂停在断点处，等待调试者调试。可以按“下一步”让程序单步执行，一条语句一条语句查看，同时变量的值也会显示出来，直至程序结束；也可以让程序调到下一个断点进行调试，如果没有下一个断点，则程序直至运行结束。
  Debug模式可以中断程序的运行，查看中断点各变量的值。Debug模式是程序开发中常用的调试技巧，是每个开发者必须具备的技能。
